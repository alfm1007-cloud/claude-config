# 6지점 동시 검증 — 성내동 CLAUDE.md 박제 정책

필드 1개 변경 시 6지점 모두 일관되게 바뀌어야 한다. "모달엔 추가했는데 목록엔 없다", "강의 컬럼이 헤더엔 있는데 데이터엔 없다" 류 사고 재발 방지.

## 사용법
사용자가 `/6point <필드명>` 또는 `/6point <필드명> in <파일패턴>` 으로 호출.

대상 필드명을 받으면 아래 6지점 자동 grep 후 표 형식으로 보고:

| 지점 | 위치 | 발견 여부 | 비고 |
|------|------|----------|------|
| 1. 입력 (모달/폼) | `index.html` `<input id="...">` | ✅/❌ | id 매칭 |
| 2. 저장 (saveXxx) | `*.js` `sb.from('...').insert/upsert` | ✅/❌ | row 객체에 필드 포함? |
| 3. 조회 (loadXxx) | `*.js` `sb.from('...').select` | ✅/❌ | select '*' 또는 명시적 컬럼? |
| 4. 표시 (목록/상세) | `*.js` `renderRow`, `tbody`, `<td>` | ✅/❌ | 헤더 colspan 일치? |
| 5. 수정 (update) | `*.js` `.update({...})` | ✅/❌ | 필드 누락 여부 |
| 6. 삭제·역방향 | 외래키, import/export, 캐시 키 | ✅/❌ | 다른 모듈 영향 |

## 자동 실행 절차
1. Grep 6번 병렬 실행 (각 지점별)
2. 발견된 위치 라인 번호 표기
3. ❌ 인 지점은 "여기를 추가/수정해야 함"으로 안내
4. 모든 ✅ 면 "6지점 통과" 토스트

## 시뮬레이션 절차 (보너스)
변경 전 머릿속으로 1회 끝까지 추적:
"사용자가 이 값을 입력 → 어떻게 흘러서 → 어디에 표시되는지"
