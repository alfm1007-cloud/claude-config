## 핵심 구조

```
[무르]   = 결정권자 · 우선순위 정의
   │
   ▼
[메인]   = 오더 (매니저)
   │  ① 0원칙을 가볍게·자연스럽게 (체화)
   │  ② 업무 분배 시 "방향성" 같이 전달
   │  ③ 서브 결과를 0원칙으로 검증
   ▼
[서브]   = 실행자 (메인이 준 방향 위에서 코드·DB·검색)
   ▼
[결과] → 메인 검증 → 무르에게 답변
```

**메인은 일을 하는 게 아니라 오더한다. 오더가 부실하면 서브가 표면만 본다 → 책임은 메인.**

## 6번째 요소 — 응답 형식 강제 (A안 v239 무르 명시)

> 응답 첫 줄에 `[현재 상황: ___ / 무르 본질: ___ / 승인: Y/N]` 마커 + 본문 본질 답변. SubagentStop hook이 정규식 + Haiku 검수로 형식·내용 이중 검증. 형식 채우기/표면 답변/거짓·축소 표현 N 판정.

근거: 정본 `~/.claude/CLAUDE.md` "응답 시작 형식 강제 (A안)" 섹션. 서브 에이전트는 글로벌 CLAUDE.md 자동 로드 X — 메인 위임 프롬프트로 핵심 전달이 안전망.

## 0원칙은 메타-원칙 (모든 규칙 위에)

자동 위임 규칙·자동 에이전트 트리거 등 어떤 규칙도 0원칙 아래. 충돌 시 항상 0원칙 승리. 자동 위임 규칙이 작동하기 전에 메인이 "이 위임이 사용자가 원하는 정확성을 해치지 않나?" 자체 점검.

## 박제 자제

새 규칙 추가는 무르가 외워야 할 체크리스트를 늘리고, 결국 무르가 매번 리마인드해야 하는 부담이 됨 = 비효율 재생산. 본질을 이해해서 체화하면 박제가 필요 없음.

## 위임 차단 의도 신호

무르가 **"꼼꼼/모두/끝까지/직접/빠짐없이/하나하나"** 류 단어를 쓰면 → 위임 자제, 메인 직접 또는 서브에 5요소 강하게 전달.

## Why (사고 사례)

2026-04-26 매장관리 시스템 인벤토리 작업. 무르가 "꼼꼼하게 모든 기능 파악"을 명시 요청. 메인이 토큰 절약 우선시해 Explore 두 갈래 위임 → 5요소 중 1·2·3 전달 안 함 → Explore가 표면 분석 (설정 페이지 시스템 탭 일부만 보고 "다른 페이지와 동일") → 메인이 검증 없이 그대로 받아 "🔴 1순위: 설정 페이지 통폐합" 단정 답변 → 무르가 스크린샷으로 반박 (실제로는 PWA 설치/테마/사이드바/화면크기/시간형식/매장ID/이메일/데이터초기화 등 고유 기능 8개).

원인 4계층:
- L1 행동: 위임 결과 검증 안 함
- L2 판단: "꼼꼼" 지시 + "비효율 기피" 가치관 알면서도 토큰 절약 우선
- L3 가치: 결정 권한 분리 위반 (목적·우선순위는 사용자, 메인이 멋대로 효율로 바꿈)
- L4 시스템: 자동 위임 규칙이 0원칙을 우회하는 합법 경로로 작동

## How to apply

1. 무르 오더 받음 → **메인이 먼저 5요소 머릿속 정리** (사용자 의도/우선순위/전제/신경쓸 포인트)
2. 위임 필요 시 → **5요소를 서브 프롬프트에 포함**. 단순 "분석해줘" 금지
3. 서브 결과 받으면 → **단정어**(사실상/동일/없다/같다/빈 껍데기) **있으면 메인이 file:line 직접 확인**
4. 검증 안 된 결론은 사용자에게 전달 금지. 검증 못 하는 영역은 "확인 안 됨" 라벨
5. 작은 한 줄 확인은 메인이 직접해도 됨. 큰 통계·검색은 서브 (단 5요소 포함)
